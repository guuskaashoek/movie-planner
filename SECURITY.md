### Security Policy

We take security seriously and appreciate responsible disclosures.

### Supported versions

This is an open-source project without formal support guarantees.  
The `main` branch and the latest tagged releases are considered supported for security fixes on a best-effort basis.

### Reporting a vulnerability

If you discover a security vulnerability in this project:

- **Do not** open a public GitHub issue with details.
- Instead, please contact the maintainer privately at:

  - **Name**: Guus Kaashoek  
  - **Email**: `guus@guuslab.com`

Please include:

- A clear description of the issue.
- Steps to reproduce.
- Any potential impact or exploit scenario you’re aware of.

You will receive an acknowledgement as soon as reasonably possible, and we will coordinate a fix and disclosure timeline with you.
